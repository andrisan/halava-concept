## Overview

Restaurant Operations enables full restaurant workflows including QR-based ordering, menu management with modifiers, kitchen queue management, and table service. Customers scan table QR codes to view menus and place orders directly from their phones (or desktop browsers).

**Note:** Consumer-facing screens (QR Menu, Order Tracking) support both mobile and desktop. Merchant operational screens (Kitchen Queue, Kitchen Display) are desktop-only due to the nature of kitchen workflows.

## Implementation Notes

### Key Components Needed

**Consumer (Mobile & Desktop):**
- `QRMenuPage` - Menu browse with cart (grid layout on desktop)
- `MenuItemCard` - Item with photo, price, quick add
- `MenuItemDetail` - Full item with modifiers (full-screen on mobile, modal on desktop)
- `ModifierGroup` - Radio/checkbox modifier options
- `SpecialInstructions` - Free-text input
- `RestaurantCart` - Current order summary
- `DineInOrderTracking` - Order status for table orders

**Merchant (Desktop Only):**
- `KitchenQueue` - Kanban-style order board
- `KitchenColumn` - New, Preparing, Ready columns
- `KitchenOrderCard` - Order summary with items
- `OrderCardDetail` - Expanded order view
- `KitchenDisplayMode` - Full-screen kitchen monitor
- `QRCodeGenerator` - Table and menu QR generation
- `TableQRCard` - Printable QR card template

### API Dependencies

**Consumer:**
- `GET /api/merchants/{id}/menu` - Menu items
- `GET /api/merchants/{id}/menu/{itemId}` - Item detail with modifiers
- `POST /api/merchants/{id}/orders` - Place dine-in order
- `GET /api/orders/{id}` - Order status

**Merchant:**
- `GET /api/kitchen/orders` - Kitchen queue
- `PATCH /api/kitchen/orders/{id}/status` - Update order status
- `POST /api/merchants/{id}/qr-codes` - Generate QR codes

### Design Tokens Required

- Colors: Order priority (age-based), status colors
- Typography: Large text for kitchen display
- Spacing: Touch-friendly kitchen buttons
- Animation: New order alert pulse
